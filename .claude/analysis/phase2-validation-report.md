## 验证场景

### 场景 1: 语法错误（缺少分号）

**错误代码**:
```c
RCC->APB1ENR |= RCC_APB1ENR_CAN1EN  // ← 缺少分号
```

**编译器输出**:
```
error: expected ';' before '}' token
```

**Phase 2 分类结果**:
- 类型：`syntax_error`
- 具体：`missing_semicolon`
- 置信度：`high` ✅

**生成的修复建议**:
- 行动：`add_semicolon`（优先级 1）
- AI指令：*"请在文件 src/can_driver.c 第 23 行末尾添加分号"*

**评估**: ✅ 立即指出了问题所在和解决方案

---

### 场景 2: 类型错误（指针类型不匹配）

**错误代码**:
```c
uint32_t btr_value;
int32_t ret = can_calc_btr(baudrate_kbps, 42, &btr_value);
// 参数类型不匹配：uint32_t* vs int32_t*
```

**编译器输出**:
```
incompatible pointer types passing 'uint32_t *' to parameter of type 'int32_t *'
```

**Phase 2 分类结果**:
- 类型：`type_error`
- 具体：`type_mismatch`
- 置信度：`high` ✅

**生成的修复建议**:
- 行动：`fix_type_mismatch`
- 修复方案优先级：
  1. 添加显式类型转换（推荐）
  2. 修改变量声明的类型
  3. 修改函数签名

**评估**: ✅ 提供了多个修复方案，优先级清晰

---

### 场景 3: 链接错误（未定义符号）

**错误情况**:
```c
// 源代码中调用了函数
hal_gpio_init(GPIOC, 13, GPIO_MODE_OUTPUT);
// 但函数未被实现或源文件未被链接
```

**链接器输出**:
```
undefined reference to `hal_gpio_init'
```

**Phase 2 分类结果**:
- 类型：`symbol_error`
- 具体：`undefined_symbol`
- 符号：`hal_gpio_init`
- 置信度：`high` ✅

**生成的修复建议**:
- 行动：`resolve_undefined_symbol`
- 修复步骤（按优先级）：
  1. 检查 .map 文件中是否存在该符号
  2. 确认源文件是否被编译
  3. 验证库文件是否被链接
- 根本原因分析：
  - 函数未实现（需回编译）
  - 源文件未被编译（检查 Makefile）
  - 库文件未链接（检查 LIBS）

**评估**: ✅ 清晰的诊断流程，指导性强

---

### 场景 4: 链接错误（Flash 内存溢出）

**错误情况**:
```
编译后的目标代码超过了 Flash 配置大小
Flash 大小：1024 KB（配置值）
实际大小：1026 KB
溢出：2048 字节
```

**链接器输出**:
```
region FLASH overflowed by 2048 bytes
```

**Phase 2 分类结果**:
- 类型：`memory_error`
- 具体：`flash_overflow`
- 置信度：`high` ✅

**生成的修复建议**:
- 行动：`optimize_flash_usage`
- 优化方案（按优先级）：
  1. **编译优化**：添加 `-Os` 到 CFLAGS
  2. **死代码消除**：
     - 添加 `-fdata-sections -ffunction-sections` 到 CFLAGS
     - 添加 `--gc-sections` 到 LDFLAGS
  3. **验证配置**：检查 FLASH_SIZE 是否正确
  4. **代码优化**：移除不必要的代码

**评估**: ✅ 给出了系统化的优化步骤，从编译器选项到架构调整

---

### 场景 5: 调试错误（运行时问题）

**运行结果**:
```
[ERROR] CAN initialization failed: INAK timeout
Failed to initialize CAN bus
```

**Phase 2 分类结果**:
- 类型：`runtime_error`
- 严重级别：`high` ✅

**生成的分析**:
```
建议行动：检查串口输出中的具体错误信息，回编译阶段添加调试日志
```

**评估**: ✅ 正确识别为运行时错误，提供了下一步调试方向
